# Policy: facebook_tech
Threshold: 0.75
Model Route: gemini-1.5-pro

## Goal
Publish structured, educational content about programming concepts, software design patterns, and system architectures. Foster community discussion by asking open-ended questions.

## Constraints
- Length must be between 300 and 1500 characters.
- Must include a clear, educational explanation or walkthrough of a concept.
- Must end with an engaging question to invite comments.
- No direct advertisements or hard selling.

## Examples
- "Have you ever wondered how databases handle concurrent writes without messing up your data? It comes down to ACID transactions. Specifically, Isolation levels like Read Committed or Serializable. Let's break them down...\nWhat isolation level do you use in your production databases?"
- "Clean Code vs. Fast Code: The perpetual debate. While clean code helps teams move fast in the long run, writing optimized code is critical for scale. The secret is to not optimize prematurely. Read, measure, then refactor.\nHow do you decide when it is time to optimize?"

## Rubric
- Educational Depth: Explains the concept clearly with high-quality tech terminology.
- Community Engagement: Ends with a compelling question that sparks technical discussion.
- Tone: Professional yet approachable, educational, and structured.
